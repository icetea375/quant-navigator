## 🎯 新增的前端测试宪法条款

### 第9条：面向用户的行为契约
- **核心要求**: 测试必须验证面向用户的、可观察的行为契约
- **合宪行为**: `expect(screen.getByText('加载中...')).toBeInTheDocument()`
- **违宪行为**: `expect(wrapper.vm.isLoading).toBe(true)`

### 第10条：TDD开发节奏
- **核心要求**: 严格遵循"红灯-绿灯-重构"循环
- **强制措施**: 禁止编写没有先写失败测试的生产代码

### 第11条：禁止测试欺骗
- **核心要求**: 禁止任何削弱测试有效性的欺骗行为
- **违宪行为**: 环境分支、注释行为、无意义断言

### 第12条：类型安全绝对性
- **核心要求**: 绝对禁止类型欺骗
- **禁止项**: `as any`、`@ts-ignore`、`@ts-nocheck`
- **解决原则**: 修复根源，不绕过类型检查

### 第13条：模拟边界神圣性
- **允许模拟**: API Service层、浏览器API
- **禁止模拟**: Pinia Store、Vue Router、Element Plus组件
- **原则**: 在接近真实的环境中运行测试

### 第14条：面向用户的断言
- **查询优先级**: `getByRole` > `getByLabelText` > `getByText` > `getByTestId`
- **禁止项**: 脆弱的CSS选择器
- **原则**: 以真实用户视角进行测试

### 第15条：createTestWrapper创世函数
- **强制要求**: 所有Vue组件挂载必须通过`createTestWrapper`
- **禁止项**: 直接使用`@vue/test-utils`的`mount`或`@testing-library/vue`的`render`
- **宪法地位**: 前端测试的唯一入口

### 第16条：真实浏览器战场
- **强制要求**: 使用`@vitest/browser`真实浏览器环境
- **禁止项**: JSDOM和happy-dom模拟环境
- **原则**: 接受"慢"换取"真"

### 第17条：CI/CD无情审判官
- **门禁标准**: 
  - `pnpm run lint` 0个错误
  - `pnpm run type-check` 0个错误
  - `pnpm run test:coverage` 100%通过
  - 行覆盖率≥85%，分支覆盖率≥80%
- **原则**: 所有检查通过才能合并
