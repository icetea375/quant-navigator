# v13.3 Monorepo结构优化计划

## 目标

实现一个无可挑剔的Monorepo结构，将架构描述修正为"模块化的单体"。

## 架构定义

**模块化的单体 (Modular Monolith)**：
- 单一部署单元，但内部按业务功能模块化组织
- 模块间通过明确定义的接口通信
- 每个模块可以独立开发、测试和维护
- 未来可以轻松拆分为微服务（如果需要）

## 当前问题分析

### 目录结构问题
1. **frontend** 和 **aigc/frontend** 在根目录，应该统一到 **packages** 下
2. **src/analysis** 在根目录，应该移动到 **packages/backend-python** 下
3. **data_services** 在根目录，应该移动到 **packages** 下
4. **backup** 目录应该移动到 **archive** 下
5. 根目录过于混乱，需要清理

### 架构描述问题
- 所有文档中描述为"微服务"，应该修正为"模块化的单体"
- 需要统一架构术语和概念

## 目标Monorepo结构

```
papa/
├── packages/                    # 所有应用包
│   ├── backend-python/         # Python模块化单体后端
│   ├── frontend-main/          # 主前端应用
│   ├── frontend-aigc/          # AIGC前端应用
│   ├── data-services/          # 数据服务包
│   └── shared-types/           # 共享类型定义
├── apps/                       # 应用入口点
│   ├── web/                    # Web应用入口
│   └── api/                    # API应用入口
├── libs/                       # 共享库
│   ├── database/               # 数据库相关
│   ├── config/                 # 配置管理
│   └── utils/                  # 工具库
├── tools/                      # 开发工具
│   ├── scripts/                # 脚本工具
│   └── tests/                  # 测试工具
├── docs/                       # 文档
├── archive/                    # 归档文件
│   ├── legacy-nestjs/          # NestJS历史代码
│   └── backup/                 # 备份文件
├── infrastructure/             # 基础设施
│   ├── docker/                 # Docker配置
│   ├── k8s/                    # Kubernetes配置
│   └── terraform/              # 基础设施即代码
└── config/                     # 全局配置
    ├── env/                    # 环境配置
    └── ci/                     # CI/CD配置
```
