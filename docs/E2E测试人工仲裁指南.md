# E2E测试人工仲裁指南

## 🎯 人工仲裁的核心职责

当E2E测试失败时，您需要判断失败的根本原因，并决定相应的行动。

## 📋 仲裁决策树

### 1. 首先查看失败信息

E2E测试失败时会显示：

```
❌ 真实世界E2E测试失败: [具体错误信息]
🔍 请人工判断失败原因:
   1. 网络连接问题？
   2. Tushare API问题？
   3. 我们的代码问题？
   4. 数据格式变化？

如果是外部问题，可以忽略此失败
如果是代码问题，请立即修复
```

### 2. 根据错误类型进行判断

#### 🔴 **网络连接问题**
**特征：**
- 错误信息包含：`ConnectionError`, `TimeoutError`, `网络超时`
- 错误信息包含：`无法连接到服务器`, `连接被拒绝`
- 测试在API调用阶段失败

**判断标准：**
- 检查您的网络连接是否正常
- 检查是否能访问其他网站
- 检查防火墙设置

**仲裁决策：**
- ✅ **忽略失败** - 这是外部网络问题
- 🔄 **重试测试** - 网络问题通常是临时的
- 📝 **记录问题** - 如果频繁出现，考虑调整超时设置

#### 🔴 **Tushare API问题**
**特征：**
- 错误信息包含：`您的token不对`, `API调用失败`, `请求被拒绝`
- 错误信息包含：`超出调用限制`, `API配额不足`
- 返回HTTP状态码：401, 403, 429, 500

**判断标准：**
- 检查TushARE_TOKEN环境变量是否正确设置
- 检查API token是否有效
- 检查是否超出API调用限制

**仲裁决策：**
- ✅ **忽略失败** - 这是API服务问题
- 🔧 **修复配置** - 如果token问题，更新配置
- ⏰ **等待重试** - 如果超出限制，等待配额重置

#### 🔴 **我们的代码问题**
**特征：**
- 错误信息包含：`TypeError`, `AttributeError`, `KeyError`
- 错误信息包含：`coroutine was never awaited`, `async/await`相关错误
- 错误信息包含：`缺少必要字段`, `数据验证失败`
- 测试在数据处理阶段失败

**判断标准：**
- 检查最近的代码变更
- 检查是否有语法错误
- 检查是否有逻辑错误

**仲裁决策：**
- 🚨 **立即修复** - 这是我们的代码问题
- 🐛 **创建Bug** - 记录问题并分配优先级
- 🔄 **回滚代码** - 如果问题严重，考虑回滚

#### 🔴 **数据格式变化**
**特征：**
- 错误信息包含：`KeyError`, `字段不存在`, `数据格式不匹配`
- 错误信息包含：`PE比率超出合理范围`, `数据验证失败`
- API返回成功，但数据结构发生变化

**判断标准：**
- 检查Tushare API文档是否有更新
- 检查返回的数据结构是否与预期一致
- 检查数据值是否在合理范围内

**仲裁决策：**
- 🔧 **更新代码** - 适配新的数据格式
- 📚 **查看文档** - 确认API变更
- 🧪 **测试验证** - 用新数据测试代码

## 🎯 仲裁原则

### 1. 保守原则
- 当不确定时，倾向于认为是外部问题
- 只有在明确确认是代码问题时，才进行修复

### 2. 效率原则
- 优先处理影响开发流程的问题
- 网络和API问题可以暂时忽略

### 3. 学习原则
- 记录每次仲裁的决策过程
- 总结常见问题模式
- 持续改进判断准确性

## 🚀 快速判断技巧

### 看错误信息关键词
- `ConnectionError`, `TimeoutError` → 网络问题
- `token`, `API`, `401`, `403` → API问题
- `TypeError`, `AttributeError`, `KeyError` → 代码问题
- `字段不存在`, `数据格式` → 数据格式问题

### 看失败阶段
- API调用阶段失败 → 网络或API问题
- 数据处理阶段失败 → 代码或数据格式问题
- 数据验证阶段失败 → 数据格式问题

### 看错误频率
- 偶尔失败 → 外部问题
- 持续失败 → 代码问题
- 特定时间失败 → 网络或API问题

## 💡 最佳实践

1. **建立判断标准**：为每种错误类型建立明确的判断标准
2. **记录决策过程**：记录每次仲裁的思考过程
3. **定期回顾**：定期回顾仲裁决策的准确性
4. **持续改进**：根据经验不断优化判断标准

记住：**人工仲裁的目标是快速、准确地判断问题根源，并采取适当的行动。**
