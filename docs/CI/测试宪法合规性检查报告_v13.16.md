## 修复建议

### 高优先级修复

#### 1. 修复存在性断言（第8条）
**目标**: 将所有`toBeDefined()`和`not.toBeNull()`替换为具体的值断言

**修复示例**:
```typescript
// 违宪代码
expect(store).toBeDefined()

// 合宪代码
expect(store.currentCaseId.value).toBeNull()
expect(store.loading.value).toBe(false)
```

#### 2. 修复用户查询（第7条）
**目标**: 使用`screen.getBy*`查询替代CSS选择器

**修复示例**:
```typescript
// 违宪代码
const loadingElement = wrapper.find('[data-testid="loading"]')

// 合宪代码
const loadingElement = screen.getByText('加载中...')
```

#### 3. 统一createTestWrapper使用（第9条）
**目标**: 移除自定义createTestWrapper，统一使用utils中的版本

### 中优先级修复

#### 1. 面向用户行为测试（第1条）
**目标**: 测试用户可见的行为而非内部状态

**修复示例**:
```typescript
// 违宪代码
expect(wrapper.vm.loading).toBe(true)

// 合宪代码
expect(screen.getByText('加载中...')).toBeInTheDocument()
```
