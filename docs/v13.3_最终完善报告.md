## 🏗️ 架构设计

### 模块化单体架构

#### 核心特征
1. **单一部署单元** - 整个应用作为一个单元部署
2. **内部模块化** - 按业务功能划分为独立模块
3. **明确接口** - 模块间通过定义良好的接口通信
4. **独立开发** - 每个模块可以独立开发、测试和维护
5. **未来可扩展** - 可以轻松拆分为微服务

#### 模块划分
- **仲裁模块** - 处理仲裁案例
- **报告模块** - 生成和管理报告
- **工作流模块** - 协调业务流程
- **LLM模块** - 提供AI能力

### 共享类型库

#### 设计原则
1. **单一事实来源** - 所有业务实体的数据结构定义都在这里
2. **类型安全** - 使用Pydantic确保运行时类型验证
3. **前后端一致** - Python后端和TypeScript前端使用相同的契约
4. **版本控制** - 所有变更都有明确的版本记录

#### 核心模块
- `common.py` - 通用类型定义
- `events.py` - 事件相关类型定义
- `reports.py` - 报告相关类型定义
- `quant_signals.py` - 量化信号相关类型定义
- `workflow.py` - 工作流相关类型定义
