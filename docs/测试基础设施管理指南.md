# 测试基础设施管理指南

## 🎯 概述

本指南详细说明测试中涉及的统一系统级要素管理，确保测试环境的**一致性**、**隔离性**和**可重复性**。

## 🏗️ 测试基础设施架构

### 1. **Docker容器化环境**

#### **服务组件**
```yaml
# docker-compose.test.yml
services:
  postgres:
    image: postgres:14
    environment:
      POSTGRES_DB: quant_navigator_test
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: testpass
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres -d quant_navigator_test"]

  redis:
    image: redis:6-alpine
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
```

#### **管理命令**
```bash
# 启动测试环境
docker-compose -f infrastructure/docker/docker-compose.test.yml up -d

# 清理并重建（确保干净环境）
docker-compose -f infrastructure/docker/docker-compose.test.yml down --volumes
docker-compose -f infrastructure/docker/docker-compose.test.yml up -d --force-recreate

# 检查服务状态
docker-compose -f infrastructure/docker/docker-compose.test.yml ps
```

## 🚀 测试执行流程

### 1. **统一执行入口**

```bash

./run-tests.sh integration   # 集成测试
./run-tests.sh e2e          # 端到端测试
./run-tests.sh all          # 所有测试
```

## 🚨 常见问题和解决方案

### 1. **Docker容器启动失败**

```bash
# 检查端口占用
lsof -i :5432
lsof -i :6379

# 清理Docker资源
docker system prune -a
docker volume prune
```

### 2. **数据库连接失败**

```bash
# 检查数据库状态
docker-compose -f infrastructure/docker/docker-compose.test.yml ps

# 检查数据库日志
docker-compose -f infrastructure/docker/docker-compose.test.yml logs postgres
```

### 3. **测试数据污染**

```bash
# 清理测试数据
./tools/tests/scripts/cleanup-test-data.js

# 重置数据库
pnpm run db:reset:test
```

### 4. **依赖版本冲突**

```bash
# 清理依赖
rm -rf node_modules
rm -rf packages/*/node_modules

# 重新安装
pnpm install
```

### 2. **测试数据缓存**

```typescript
// 测试数据缓存
const testDataCache = new Map();

export function getCachedTestData(key: string, factory: () => any) {
  if (!testDataCache.has(key)) {
    testDataCache.set(key, factory());
  }
  return testDataCache.get(key);
}
```
