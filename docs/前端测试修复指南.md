
# 前端测试修复指南

## 概述
本指南记录了前端测试文件中的常见问题及其修复方法，提供了21种修复模式和批量修复工具，帮助快速解决测试问题。

## 修复流程
请严格遵守下面步骤：
1、检查前端测试文件是否还有问题。选一个有问题的一个文件，找到其所有问题后，然后进行修复。
2、修复完后，更新所有所有新发现的模型到tools/scripts/batch-fix-frontend-tests.py，运行tools/scripts/batch-fix-frontend-tests.py来修复其他文件。（必须做，必须是所有新发现的模型）
3、用上述脚本批量修复后，更新本文档，记录新增的修复模型。
4、挑选下一个文件，进行修复。直到所有前端测试文件的错误都修复为止。


## **🔧 已建立的修复模式**

### **模式1：SupportedEventInterface绕行**
- **问题**: `trigger()`调用导致`SupportedEventInterface is not a constructor`
- **解决**: 使用`wrapper.vm.methodName()`或`wrapper.vm.$emit()`绕行
- **适用**: 所有有事件触发的测试
- **示例**: ArbitrationCaseList.test.ts

### **模式2：Mock数据完善**
- **问题**: 测试期望的数据字段在mock中缺失
- **解决**: 检查组件模板使用的字段，确保mock数据包含所有必要字段
- **适用**: 所有使用mock数据的测试
- **示例**: components.test.ts (Auth组件username字段, Market组件briefing数据)

### **模式3：重复配置合并**
- **问题**: 多个`global`配置块导致语法错误
- **解决**: 合并为一个`global`块，确保包含`plugins: [pinia]`
- **适用**: 所有有重复配置的测试文件
- **示例**: ArbitrationCaseList.test.ts, FlowAndChipsViewer.test.ts

### **模式4：组件数据设置**
- **问题**: 组件渲染失败，`exists()`返回false
- **解决**: 设置`wrapper.vm.data`、`wrapper.vm.loading`、`wrapper.vm.error`等状态
- **适用**: 所有有组件渲染问题的测试
- **示例**: RawTextExplorer.test.ts, FlowAndChipsViewer.test.ts

### **模式5：Vue Router导入问题**
- **问题**: `Failed to resolve import "vue-router"`
- **解决**: 注释掉vue-router导入，使用mock对象替代
- **适用**: 所有使用vue-router的测试文件
- **示例**: Home.test.ts, MarketRadar.test.ts

### **模式6：重复声明修复**
- **问题**: `The symbol "pinia" has already been declared`
- **解决**: 删除重复的变量声明
- **适用**: 所有有重复声明的测试文件
- **示例**: arbitration-flow.test.ts

### **模式7：空文件清理**
- **问题**: `No test suite found in file`
- **解决**: 删除空的测试文件
- **适用**: 所有空的测试文件
- **示例**: pinia-heart-surgery.test.ts

### **模式8：语法错误修复**
- **问题**: 多个地方缺少闭合括号导致语法错误
- **解决**: 检查所有mount()调用，确保global配置和props配置正确闭合
- **适用**: 所有有语法错误的测试文件
- **示例**: PersonalPrecedentViewer.test.ts

### **模式9：批量修复脚本**
- **问题**: 多个文件有相同的修复模式需要应用
- **解决**: 创建Python脚本批量应用所有修复模式
- **适用**: 所有需要批量修复的测试文件
- **示例**: 修复了22个文件，应用了所有8个修复模式
- **脚本**: `tools/scripts/batch-fix-test-patterns.py`

### **模式10：Mock数据字段名匹配**
- **问题**: 组件期望的字段名与mock数据中的字段名不匹配
- **解决**: 检查组件模板使用的字段，确保mock数据包含正确的字段名
- **适用**: 所有使用mock数据的测试
- **示例**: FinancialSnapshot.test.ts (revenueCagr3y vs revenue3YearCAGR, dataCompletenessScore vs dataCompleteness)

### **模式11：测试期望与实际渲染内容匹配**
- **问题**: 测试期望检查的数据格式与组件实际渲染的格式不匹配
- **解决**: 根据组件实际渲染的内容调整测试期望
- **适用**: 所有有数据渲染问题的测试
- **示例**: FinancialSnapshot.test.ts (期望数据值但实际渲染趋势百分比)

### **模式12：空状态组件Mock**
- **问题**: el-empty组件在测试中没有正确渲染
- **解决**: 在stubs中添加el-empty组件的mock
- **适用**: 所有有空状态显示的测试
- **示例**: FinancialSnapshot.test.ts

### **模式13：直接操作组件状态**
- **问题**: `SupportedEventInterface is not a constructor`导致事件触发失败
- **解决**: 直接设置组件内部状态而不是触发事件
- **适用**: 所有有表单交互的测试
- **示例**: ArbitrationDecisionDialog.test.ts
- **代码**: `wrapper.vm.form.decision = 'reject'` 替代 `await radio.setChecked()`

### **模式14：表单字段名匹配**
- **问题**: 测试中使用的字段名与组件实际字段名不匹配
- **解决**: 检查组件定义，使用正确的字段名
- **适用**: 所有有表单字段访问的测试
- **示例**: ArbitrationDecisionDialog.test.ts (formData vs form, reason vs reasoning)

### **模式15：表单验证模拟**
- **问题**: `handleSubmit`方法需要表单验证通过才触发事件
- **解决**: 模拟表单验证通过
- **适用**: 所有有表单提交的测试
- **示例**: ArbitrationDecisionDialog.test.ts
- **代码**: 添加`mockValidate`模拟表单验证

### **模式16：Mock组件内容渲染**
- **问题**: Mock组件没有渲染期望的文本内容
- **解决**: 在stubs中添加template包含正确的文本内容
- **适用**: 所有有自定义组件Mock的测试
- **示例**: DataPanelContainer.test.ts
- **代码**: `RawTextExplorer: { template: '<div class="raw-text-explorer-mock">原始文本浏览器</div>', props: ['data', 'loading', 'error'] }`

### **模式17：直接调用组件方法**
- **问题**: 子组件事件无法正确触发父组件处理
- **解决**: 直接调用父组件的事件处理方法
- **适用**: 所有有事件传递的测试
- **示例**: DataPanelContainer.test.ts
- **代码**: `wrapper.vm.handleToggleMaximize('raw-text')` 替代 `component.trigger('toggle-maximize')`

### **模式18：el-empty组件Mock修复**
- **问题**: el-empty组件没有正确显示description属性
- **解决**: 修复el-empty组件的mock模板，确保显示description内容
- **适用**: 所有有空状态显示的测试
- **示例**: PersonalPrecedentViewer.test.ts, RawTextExplorer.test.ts
- **代码**: `template: '<div class="el-empty">{{ description }}</div>'` 替代 `<slot />`

### **模式19：直接调用组件事件处理方法**
- **问题**: `SupportedEventInterface is not a constructor`导致事件触发失败
- **解决**: 直接调用组件的事件处理方法而不是触发click事件
- **适用**: 所有有事件处理的测试
- **示例**: ArbitrationToolbar.test.ts
- **代码**: `wrapper.vm.handleRefresh()` 替代 `refreshButton.trigger('click')`

### **模式20：修复click事件触发问题**
- **问题**: `click is not defined`或错误的`await wrapper.vm.$emit(click)`
- **解决**: 修复错误的click事件触发语法
- **适用**: 所有有click事件触发的测试
- **示例**: ArbitrationToolbar.test.ts, FinancialSnapshot.test.ts
- **代码**: `await refreshButton.trigger('click')` 替代 `await wrapper.vm.$emit(click)`
