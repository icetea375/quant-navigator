# 测试架构重构方案

## 问题分析

当前测试文件集中在 `tools/tests/` 导致的问题：

1. **依赖包版本冲突** - 不同包使用不同版本的依赖
2. **模块路径映射复杂** - 需要复杂的 `moduleNameMapper` 配置
3. **维护困难** - 路径错误难以调试和修复
4. **测试覆盖率分散** - 难以统计各包的覆盖率

## 推荐方案：分布式测试架构

### 目标架构

```
packages/
├── backend-python/
│   ├── src/
│   ├── tests/                    # 后端测试
│   │   ├── unit/
│   │   │   ├── services/
│   │   │   ├── api/
│   │   │   └── schemas/
│   │   ├── integration/
│   │   │   ├── api/
│   │   │   └── services/
│   │   ├── conftest.py
│   │   └── __init__.py
│   └── package.json              # 包含测试依赖
├── frontend-main/
│   ├── src/
│   ├── tests/                    # 前端测试
│   │   ├── unit/
│   │   │   ├── components/
│   │   │   ├── stores/
│   │   │   └── utils/
│   │   ├── integration/
│   │   │   ├── components/
│   │   │   └── pages/
│   │   ├── setup.ts
│   │   └── __init__.py
│   └── package.json              # 包含测试依赖
└── shared-types/
    ├── src/
    ├── tests/                    # 共享类型测试
    │   ├── unit/
    │   └── __init__.py
    └── package.json

tools/
└── tests/
    ├── integration/              # 跨包集成测试
    │   ├── frontend-backend/
    │   ├── api-gateway/
    │   └── e2e-workflows/
    ├── e2e/                     # E2E测试
    │   ├── user-journeys/
    │   └── critical-paths/
    └── config/                  # 测试配置
    │   ├── jest.config.js
    │   ├── playwright.config.ts
    │   └── test-setup.ts
```

### 优势

1. **依赖隔离** - 每个包管理自己的测试依赖
2. **路径简单** - 测试文件与源码在同一包中
3. **维护容易** - 测试与代码同步维护
4. **覆盖率清晰** - 每个包有独立的覆盖率报告
5. **符合测试宪法** - 遵循"测试即契约"原则

### 实施步骤

#### 阶段1：创建分布式测试结构

1. **为每个包创建测试目录**
   ```bash
   mkdir -p packages/backend-python/tests/{unit,integration}
   mkdir -p packages/frontend-main/tests/{unit,integration}
   mkdir -p packages/shared-types/tests/unit
   ```

2. **更新各包的 package.json**
   - 添加测试脚本
   - 添加测试依赖
   - 配置测试环境

3. **移动现有测试文件**
   - 单元测试 → 各自包中
   - 集成测试 → 各自包中
   - 跨包集成测试 → tools/tests

#### 阶段2：配置测试环境

1. **各包独立配置**
   - Jest/Vitest 配置
   - 测试环境设置
   - 覆盖率配置

2. **根目录统一管理**
   - 全局测试脚本
   - 覆盖率聚合
   - CI/CD 集成

#### 阶段3：验证和优化

1. **运行测试验证**
2. **检查覆盖率**
3. **优化测试用例**
